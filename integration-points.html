Integration Points

Integration points are services that are injected into Integration Jobs.

Available integration points are "Elasticsearch," "RabbitMQ," "JsonService," and "Mail." They are configured in the IntegrationEngine.json file.

To create an integration point, create a class that extends an existing integration point class. Note that an "integration point class" is one that implements IntegrationEngine.Core.IntegrationPoint.IIntegrationPoint. Built-in integration point classes are the following.

  • IntegrationEngine.Core.Elasticsearch.ElasticClientAdapter
  • IntegrationEngine.Core.Mail.MailClient
  • IntegrationEngine.Core.MessageQueue.RabbitMQClient
  • IntegrationEngine.Core.ServiceStack.JsonService

To configure the integration point, the IntegrationPointName, defined in the configuration file, has to be the same name as the integration point's class name. For example, in the following code snippet the class name is "MyMailClient" and, in the config sample shown below the class, the IntegrationPointName is "MyMailClient" as well. 

using IntegrationEngine.Core.Mail;

namespace MyProject.MyIntegrationPoints
{
    public class MyMailClient : MailClient
    {
    }
}
{
    // ...
    "IntegrationPoints": {
        "Mail": [
            {
                "IntegrationPointName": "MyMailClient", // Matches the class name.
                "HostName": "localhost",
                "Port": 25
            }
        ]
    }
    // ...
}
